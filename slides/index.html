The URL Working Group

Advisory Board Edition

plh@w3.org / rubys@intertwingly.net

Purpose

This presentation focuses on a proposal for a URL Working Group at W3C.

Current situation

  • URL work is done at the WHATWG, by Sam and Anne
  • WebApps has URL in its charter but isn't really involved
  • The browser implementors aren't involved either

URL Working Group

  • New approach to Web Architecture-related documents
  • Focused narrow scope
  • Use W3C Software license
  • Coordinate with IETF, WHATWG, TAG, I18n, and others

See draft charter.

Communication

  • public-url@w3.org
  • An issue list is maintained at WebPlatform.org
  • A discourse category is made available on an experimental basis.

See work mode (to be updated if separate WG).

Decision Policy

  • Seek to make decisions when there is consensus and with due process
  • Based on proposals by editors or other participants
  • Calls for Consensus only for process related items

Participation

Expected to have 5 or more active participants

  • Needs to have at least three distinct browser-engine project participants
  • Needs an updated Invited Expert and Collaborator Agreement

Next steps

  1. MUST get participation commitments from browser-engine projects
  2. Identify Chair for the Group
  3. Get W3C to update its IE agreement
  4. Start AC review of the charter as well as getting IETF feedback

Alternatives

If the Group can't be set up by end of March:

  1. The Director permits HTML 5.1 to normatively link to WHATWG URL.
  2. or, The Director permits HTML 5.1 to normatively link to a URL from a Community Group

Thank you!
